What We're Building

Follow a lead developer through the full AIDLC workflow as they build a brand-new Notification Service from scratch — from PO intent statement to Jenkins deployment.

The Business Requirement (from Product Owner)

📋 Azure DevOps — Work Item #5012 (from PO)
EPIC #5012 · Product Owner: Sarah T. · Priority: High
Build a real-time notification service for the platform
Context: Currently, users have no way to receive real-time alerts about events in the platform (invoice ready, payment failed, subscription expiring). This is blocking enterprise sales.

Request: Create a new notification service that can send email and in-app notifications triggered by platform events via Azure Service Bus. Must support templates, user preferences, and unsubscribe.

Constraints: Use SendGrid for email. Must work within our Azure AD security model. No notifications for users who have opted out per preference.

Success: 99% email delivery within 30 seconds of event. Zero false notifications.

Based on requirements and your semantic context (.NET 8, CQRS, Repository pattern), I'm proposing this architecture for notification-service:

Solution Structure:
  • NotificationService.API — REST endpoints, Azure AD auth, user preferences
  • NotificationService.Worker — Azure Service Bus consumer, processes events
  • NotificationService.Core — Domain, commands/queries via MediatR
  • NotificationService.Infrastructure — EF Core, SendGrid, Service Bus client
  • NotificationService.Tests — xUnit, NSubstitute, FluentAssertions

Open Decision OD-01 resolution: I recommend SQL Server with table partitioning by CreatedDate for notification history. Rationale: consistency with your existing stack (avoids a new Azure storage dependency), partitioning handles the 90-day retention efficiently.

⚠️ Question for Priya: Should the Worker use a Service Bus Subscription (fan-out to multiple consumers) or a Queue (single consumer)? The subscription model supports scaling but requires topic configuration on the platform-events bus.
Priya (Architect)
Use subscription — we'll need fan-out when we add audit logging later. I approve the overall architecture and OD-01 resolution. Proceed with code generation.
