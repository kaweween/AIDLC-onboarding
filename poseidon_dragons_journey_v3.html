Call to action21 / 23
If a non-developer built this in 8 days,
where are you on the curve?
Steve Yegge's 8 Levels of Agentic Engineering — and a realistic upgrade path for developers.
Level 1
Tab completion only
Copilot autocomplete. Occasionally ask ChatGPT a question. Zero agentic use.
Level 2
Agent in IDE, supervised
Cursor/Copilot sidebar. Agent asks permission for every tool call. You review everything carefully.
Level 3
Agent in IDE, YOLO mode
Permissions off. Agent runs freely. You review diffs, not every step.
Level 4
Wide agent fills the IDE
Agent owns the screen. You're reading diffs and steering direction, not writing code.
Level 5 ★
CLI, single agent, YOLO
Claude Code in terminal. No IDE. Diffs scroll. You are directing, not coding. This is where Yegge says most devs should be targeting now.
Level 6
CLI, 3–5 parallel agents
Multiple Claude Code instances running simultaneously. You are multiplexing between them. Very fast.
Level 7
10+ agents, hand-managed
Pushing the limits of manual coordination. You start asking: how do I orchestrate all of these?
Level 8
Build your own orchestrator
Agents coordinating agents. You are an engineering director for a team of AI. Gas Town territory.
Yegge's warning: 70% of engineers are still at Level 1–2. The gap between Level 2 and Level 6 is the difference between slow and 10–100× faster. You will not be replaced by AI. You will be replaced by a developer using AI better than you.
Call to action22 / 23
Your upgrade path. Start this week.
WEEK 1–2
Level 1 → 3 · Get an agent, turn off the leash
Install Claude Code or Cursor. Pick one real task — not a toy. Let the agent run. Review the diff, not every step. Accept that it won't be perfect. Ship it anyway and learn what to fix.
MONTH 1
Level 3 → 5 · Move to the CLI, write requirements not code
Switch to Claude Code in terminal. Write specs and user stories for your agent instead of code. Add tests — they are your safety net when you stop reading every diff. This is where Kareen's AIDLC workflow lives.
MONTH 2–3
Level 5 → 6 · Run agents in parallel, multiply your output
Open 3–5 Claude Code sessions simultaneously on different features or branches. You are now a tech lead managing a small AI team. Your bottleneck shifts from writing to deciding.
ONGOING
Invest in the skills that AI can't replace
Domain knowledge · System design · Writing clear requirements · Knowing when something is wrong · Product judgment. These are what make you the irreplaceable human in the loop. AI amplifies them — it cannot substitute for them.
